## Simplification Patterns

### Pattern 1: Extract Report Template
**Used for**: All review agents, validation skills

**Steps**:
1. Extract report format (lines 100-350 typically) to `.aknakos/templates/reports/{name}-report.md`
2. Replace in agent/skill with: "Use template: `.aknakos/templates/reports/{name}-report.md`"
3. Keep only decision criteria in main file

**Example**: test-reviewer (505 â†’ 91 lines)

### Pattern 2: Extract Examples
**Used for**: All conversational skills, workflow skills

**Steps**:
1. Identify example sections (usually 100-300 lines)
2. Move to `.aknakos/examples/{category}/{name}-example.md`
3. Replace with: "See `.aknakos/examples/{category}/{name}.md` for detailed example"
4. Keep 1 critical inline example if needed

**Savings**: 100-300 lines per file

### Pattern 3: Convert to Imperative
**Used for**: All files

**Replacements**:
- "This skill helps you..." â†’ "Generate X by..."
- "The process involves..." â†’ "1. Load X. 2. Execute Y."
- "You should consider..." â†’ "Consider X when Y."
- "This agent is designed to..." â†’ "Verify X. Check Y."

**Remove**:
- "What This Does" sections (redundant with description)
- "When to Use" sections (redundant with description)
- "Integration with Workflow" (unless unique)
- "Related Skills/Agents" (redundant with whats-next)

**Savings**: 50-150 lines per file

### Pattern 4: Condense Process Steps
**Used for**: Workflow skills with verbose step-by-step instructions

**Steps**:
1. Convert multi-paragraph explanations to bullet points
2. Remove redundant context/justification
3. Keep "what" and "why", remove "how it works" explanations
4. Use tables for comparison/options

**Savings**: 100-200 lines per file
